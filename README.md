# Web UI Automation Sandbox

このリポジトリは、PlaywrightとPythonを用いた様々なテスト自動化パターンの習得を目的としたプロジェクトです。
実務で通用する「堅牢でメンテナンスしやすいテストコード」のベストプラクティスを蓄積しています。

## 🛠 技術スタック
- **Language:** Python 3.12.6
- **Framework:** pytest, pytest-playwright
- **Library:** Playwright (Chromium)

## 💡 技術的なこだわりとQA視点
1. **データ駆動テスト (Data-Driven Testing)**
   - `pytest.mark.parametrize` を採用し、テストロジックとテストデータを分離。
   - 同値分割法や境界値分析の考え方をベースにデータセットを定義し、最小限のコードで複数のシナリオ（ユーザー名間違い、パスワード間違い等）を検証しています。

2. **オートウェイト（自動待機）の活用**
   - 固定待ち（sleep）を排除し、Playwrightの `expect` によるオートウェイト機能を最大限に活用。
   - 実行環境の速度に左右されない、不安定なテスト（Flaky Test）を防止する堅牢なスクリプト設計を行っています。

3. **セマンティックな要素特定**
   - CSSセレクタだけでなく、`get_by_role` などのユーザー視点に近いロケーターを優先的に使用し、UI変更に強く、可読性の高いコードを記述しています。
